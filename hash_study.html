HASH : 대칭키 암호화


STEP 1




해시 함수는 입력 데이터를 고정된 길이의 문자열이나 숫자로 변환하는 일방향 암호화 함수입니다. 해시 함수는 데이터를 짧은 "해시 값"으로 변환하지만, 그 해시 값을 다시 원래의 데이터로 되돌릴 수는 없습니다. 주로 데이터 무결성 확인이나 암호화된 비밀번호 저장 등에 사용됩니다.
중요한 특징은 같은 입력 데이터는 항상 같은 해시 값을 생성하지만, 입력 데이터가 조금이라도 달라지면 완전히 다른 해시 값이 생성된다는 점입니다. 쉽게 생각하자면, 해시 함수는 00이가 상자에 넣은 물건을 간단한 라벨로 바꾸는 과정이라고 보면 됩니다. 상자에 사과를 넣으면 그 상자에 "001"이라는 라벨이 붙는 것처럼 말입니다. 상자를 다시 열어봐도 이 라벨만 보고는 안에 뭐가 들어있는지 알 수 없습니다.
하지만 똑같은 사과를 넣으면 항상 "001"이라는 같은 라벨이 붙습니다.




STEP 2




해시 함수는 주로 데이터 무결성 확인, 암호화된 비밀번호 저장, 그리고 디지털 서명 등에 사용됩니다. 해시 함수의 특징인 일방향성과 고정된 출력 길이 덕분에 보안적인 활용도가 매우 높습니다. 대표적으로 해시 함수가 사용되는 분야를 살펴보겠습니다.



- 파일 무결성 확인
해시 함수는 파일이나 데이터의 무결성을 확인하는 데 주로 사용됩니다. 파일을 다운로드한 후 그 파일의 해시 값을 비교해서 원본 파일과 일치하는지 확인할 수 있습니다.
예를 들어, 소프트웨어 파일을 다운로드할 때 해시 값을 제공하는 경우가 많은데, 다운로드한 파일의 해시 값을 비교해서 파일이 변조되지 않았는지 확인할 수 있습니다.

- 비밀번호 저장
해시 함수는 비밀번호를 암호화된 형태로 저장할 때 사용됩니다. 사용자가 입력한 비밀번호를 그대로 저장하지 않고, 해시 값을 저장해 보안을 강화하는 방식입니다.
예를 들어, 회원가입 시 입력한 비밀번호를 해시 함수로 변환한 후 저장하면, 데이터베이스에 있는 해시 값만으로는 원래 비밀번호를 알 수 없습니다.

- 디지털 서명 및 인증
해시 함수는 디지털 서명에서 중요한 역할을 합니다. 데이터를 해시 함수로 변환한 후 그 값을 서명하고, 이를 통해 데이터의 무결성과 서명을 확인할 수 있습니다.
데이터가 조금이라도 수정되면 해시 값이 완전히 달라지기 때문에, 데이터가 변조되지 않았음을 확인하는 데 사용됩니다.

- 파일 다운로드
해시 함수는 소프트웨어나 파일을 다운로드할 때 파일의 무결성을 확인하는 데 많이 사용됩니다. 다운로드한 파일이 원본 파일과 같은지 확인하는 데 해시 값이 중요한 역할을 합니다.

- 비밀번호 관리
대부분의 웹사이트와 시스템에서는 사용자의 비밀번호를 그대로 저장하지 않고, 해시 값으로 변환한 후 저장합니다. 이를 통해 데이터베이스가 해킹당하더라도 원본 비밀번호를 알 수 없게 만들어 보안을 강화합니다.
블록체인

- 해시 함수는 블록체인 기술에서도 필수적인 역할을 합니다. 각 블록에 포함된 데이터를 해시 값으로 변환해, 블록 간의 연결성을 보장하고 데이터 변조를 방지합니다. 해시 함수는 데이터를 원본으로 복구할 수 없기 때문에 일방향성을 가지고 있습니다. 즉, 한 번 해시된 데이터를 다시 원래 상태로 되돌릴 수 없습니다. 이 때문에 해시 함수는 주로 데이터의 무결성 확인이나 비밀번호 보호 같은 상황에서 강력한 보안 기능을 제공합니다. 그리고 아주 큰 데이터도 고정된 길이의 해시 값으로 변환할 수 있기 때문에, 효율적이면서도 안전하게 데이터를 다룰 수 있습니다.





STEP 3



해시 함수의 작동 방식을 상자와 라벨로 쉽게 설명하겠습니다. 이 방식은 입력 데이터를 간단한 라벨로 바꾸고, 그 라벨만 남기는 작업을 상상하면 됩니다.



1. 상자에 라벨 붙이기 - 해시 값 생성
데이터를 해시 함수에 넣으면 고정된 길이의 해시 값으로 변환됩니다. 이 해시 값은 원본 데이터에서 추출되지만, 다시 원본 데이터를 복구할 수는 없습니다. 즉, 상자에 사과를 넣으면 "001"이라는 라벨이 붙고, 바나나를 넣으면 "002"라는 라벨이 붙는 것입니다.
라벨만 보면 안에 뭐가 있는지 알 수 없지만, 같은 물건을 넣으면 항상 같은 라벨이 붙습니다.


2. 상자에 조금이라도 다른 물건 넣으면 완전히 다른 라벨 붙이기
만약 상자 안에 있는 물건이 조금이라도 다르면, 라벨도 완전히 다른 것으로 바뀌게 됩니다. 사과 대신 사과에 한 조각만 더 추가해도, 아예 새로운 라벨이 붙습니다.
입력 데이터가 조금이라도 달라지면 해시 값은 완전히 다른 값으로 변환됩니다. 이 때문에 데이터 무결성 확인에 사용되기도 합니다.


3. 라벨로 데이터 확인하기 - 데이터 무결성 확인
해시 함수는 데이터를 간단한 라벨로 변환하지만, 이 라벨을 통해 데이터가 변조되지 않았는지 확인할 수 있습니다. 원본 데이터와 동일한 입력이 들어가면, 같은 라벨이 생성되기 때문입니다.
해시 값을 사용하면 데이터를 비교하거나 무결성을 확인할 수 있습니다. 만약 데이터가 조금이라도 변경되었다면 해시 값이 달라지기 때문에, 원본 데이터가 변조되지 않았는지 확인할 수 있습니다. 즉, 다운로드한 파일이 올바른 파일인지 확인하고 싶을 때, 원본 파일과 동일한 라벨이 붙어 있으면 그 파일이 변조되지 않았다는 걸 알 수 있는 것입니다.
